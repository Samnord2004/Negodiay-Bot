# 🏕️ Руководство по деплою ИИ-ассистента «Максимка» на Bothost / VPS

Этот проект разработан как полнофункциональное full-stack приложение, включающее интерактивный дашборд организатора на React/Vite и высокопроизводительный сервер на Node/Express для интеграции ИИ-бота с корпоративным мессенджером **МАКС**.

Благодаря встроенной сборке esbuild вся серверная часть приложения упаковывается в один компактный и готовый к запуску файл, что идеально упрощает развертывание на **Bothost** или на любом классическом VPS.

---

## 🛠️ Шаг 1. Переменные окружения (Environment Variables)

В панели управления вашего хостинга (например, в реестре переменных Bothost) укажите следующие ключи:

1. **`GEMINI_API_KEY`**
   * **Где взять:** В личном кабинете Google AI Studio.
   * **Зачем:** Обеспечивает работу нейросетевых моделей Gemini, за счет которых бот делает детальный психоанализ участников похода («Негодяев»), распознает должников и гибко подстраивает свой лексикон.

2. **`NODE_ENV`**
   * **Значение:** `production` (активирует оптимизированный корпоративный режим обслуживания без лишних логов отладки).

3. **`PORT`**
   * **Значение:** `3000` (стандартный входящий порт, принимающий запросы от мессенджера МАКС).

---

## 🏗️ Шаг 2. Настройки сборки и запуска

При привязке репозитория на хостинге выберите шаблон **NodeJS** и укажите следующие параметры:

* **Команда установки зависимостей (Install Command):**
  ```bash
  npm install
  ```
* **Команда сборки (Build Command):**
  ```bash
  npm run build
  ```
  *(Скрипт компилирует и минифицирует фронтенд, а также собирает серверную часть в один плоский файл `dist/server.cjs`)*

* **Команда запуска (Start Command / Entrypoint):**
  ```bash
  npm run start
  ```
  *(Это запустит команду `node dist/server.cjs` в автономном фоновом режиме, обеспечивая раздачу интерактивного веб-интерфейса и работу API)*

---

## 🔌 Шаг 3. Интеграция с мессенджером МАКС (MAX Messenger API)

Интеграция ИИ-помощника в корпоративный рабочий чат **МАКС** выполняется по стандартной схеме вебхуков через единый серверный REST API:

### Endpoint: `POST https://<your-app-domain>/api/bot-respond`

#### Формат запроса (JSON):
```json
{
  "message": "Максимка, привет! Кто там у нас по сборам задолжал?",
  "senderName": "Александр",
  "senderNickname": "alex_sing",
  "swearingLevel": "medium"
}
```

#### Формат ответа (JSON):
```json
{
  "text": "Алексей, Андрюха Хорёк и Иришка еще не сдали! Саня, пинай их сильнее, у нас плов и сплав горят!",
  "detectedPsychotype": "Весельчак-балагур",
  "detectedPsychotypeExplanation": "Слишком много песен поет и позитив наводит.",
  "adapterStyleUsed": "Угарный походник"
}
```

Благодаря сквозной синхронизации, все сообщения, отправленные пользователями через внешние каналы МАКС, мгновенно учитываются в едином веб-интерфейсе организатора похода, позволяя планировать меню костра, закупку продуктов, список задач и графики расходов в реальном времени!
